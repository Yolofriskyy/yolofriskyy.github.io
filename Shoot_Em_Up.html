Shoot 'Em Up

Endless multidirectional platform shooter, small prototype to test randomized platform generation on.

Shoot 'Em Up Gameplay

Overview

Shoot 'Em Up is a 2D endless runner platform shooter prototype, challenging the player to survive and achieve the longest possible distance. The player gains speed boosts by defeating enemies, which are crucial for surviving random events as the game progresses

Core mechanics

Physics based movement via coroutines

Fluid movement with coroutines

Apart from platform generation, most of the code in this project doesn’t need to run every frame. Specifically, code like player movement is designed to update frame-by-frame only when the player executes an input direction. To improve performance, I’ve implemented coroutines instead of relying on condition checks in the update function.

Multidirectional randomized platform generation

Multidirectional randomized platform generation

How it works

Three platforms randomly chosen from an array of premade platforms are always present as the character is moving, one to the left, middle and right of the character. Player distance is measured from a point (spawn point), after a certain amount of units + other factors travelled, a platform is instantiated in the direction of the outermost platform; for example if the player travels from the middle platform to the right platform, once distance is met a new platform will spawn in that direction.

Other factors such as distance between each platform, player height and platform index are also taken into account before a new platform is generated.

Multidirectional randomized platform generation

Death wall event

Random world events - Death Wall

As the player progresses, world events randomly take place. Moving death wall forces the player to move the opposite direction, showcasing the need to move and generate platforms in both directions. The players goal is to achieve the furthest distance possible, death wall counteracts that as moving the opposite direction subtracts from the players current distance

The further the distance travelled, the faster the death wall gets. Players need to make use of the speed multiplier from killing obstacles, so they can outrun the death wall at further distances.

Speed kill multiplier

Speed multiplied per kill

Each enemy kill grants a speed stack, adding 0.1x to the player’s base speed. The multiplier bar (displayed on the right) then begins to deplete exponentially, removing one stack at a time until the multiplier resets to 1x. A new kill refills the bar and resets its depletion rate.
